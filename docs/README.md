## Editorial Rules

- Every Markdown document must include a date.
- Public docs must avoid secrets, raw private keys, JWT signing material, Supabase service-role keys, generated API keys, private operator commands, or local log excerpts.
- Operational runbooks and deployment checklists belong in private/operator repositories, not this public documentation repo.
- Internal planning notes belong outside the public docs tree.
- Formal artifacts may be published only when their source date, byte length, and hash are recorded.
